环境配置

有多种方式可以搭建K8s的开发环境，其中最常见的两种方式分别是Minikube (docs) 和 docker-desktop。Minikube在多个虚拟机上（通过VirtualBox或类似的东西）配置了本地K8s集群。而docker-desktop是在docker容器中配置了K8s集群。

使用Docker Desktop搭建本地K8s集群

在这个教程中，我们所有的安装都是基于Docker Desktop。我们甚至可以使用具有相同功能且影响较小的轻量级minikube来安装K8s集群。

安装

在你的电脑上下载并安装好 Docker-Desktop 。最新版本已经内嵌了一个K8s版本，所以不需要再额外进行安装。只需要按照 此处 文档的说明获取K8s并在docker-desktop中运行就可以了。一旦K8s在docker-desktop中启动成功，你就可以看到如下信息：

这意味着：

  1. K8s已经启动。
  2. kubctl 命令已经在/usr/local/bin目录安装好了
  3. K8s的上下文已经在docker-desktop中设置好了。

问题：

由于国内的网络下不能很方便的下载 Kubernetes 集群所需要的镜像, 导致集群启用失败。这里提供了一个简单的方法, 利用 GitHub Actions 实现 k8s.gcr.io 上 kubernetes 依赖镜像自动同步到 Docker Hub 上指定的仓库中。 通过 load_images.sh 将所需镜像从 Docker Hub 的同步仓库中取回，并重新打上原始的tag。镜像对应关系文件可以查看: images。具体可以参照Docker Desktop for Mac 开启并使用 Kubernetes。

访问远程K8s集群

下面的设置假定你已经安装了远程Kubernetes集群。有关如何访问多个集群并将其集成为一般视图，请遵循K8s 的访问多个集群文档。或按照以下简化步骤操作：

  1. 从远程集群中获取K8s config文件，将其复制到本地计算机，并为其指定一个唯一名称，例如：config-remote

  2. 保存KUBECONFIG环境变量（如果已设置）

    xxxxxxxxxx
    export KUBECONFIG_SAVED=$KUBECONFIG

  3. 将新文件添加到环境变量

    xxxxxxxxxx
    export KUBECONFIG=$KUBECONFIG:config-remote

  4. 运行命令kubectl config view来检查上面两个配置是否能访问

  5. 使用 kubectl config use-context my-remote-cluster命令切换上下文

  6. 确认当前上下文现在已切换到远程集群配置:

    xxxxxxxxxx
    kubectl config get-contexts
    CURRENT   NAME                   CLUSTER                      AUTHINFO             NAMESPACE
              docker-for-desktop     docker-for-desktop-cluster   docker-for-desktop
    *         my-remote-cluster      kubernetes                   kubernetes-admin
