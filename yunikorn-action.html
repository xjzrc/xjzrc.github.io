用户手册

在读这个手册之前，我们假设你已经有一个K8s集群或者一个k8s本地的开发环境（e.g MiniKube）。也假设你已经正确的安装好了kubectl命令。根据这个 手册 可以使用 docker-desktop 去搭建一个k8s的本地集群。

在这个用户手册中，所用提到的文件都是 yunikorn-k8shim 仓库的一部分。它们都在仓库的 deployments 子目录中。下面所用执行的命令都必须在deployments子目录下。

快速开始

安装helm命令

xxxxxxxxxx
brew install kubernetes-helm

在 K8s 集群中部署 YuniKorn 最容易的方式就是使用 helm charts。

xxxxxxxxxx
helm install ./yunikorn

它首先会创建一个configmap 来存储有关YuniKorn的配置信息，然后将YuniKorn scheduler和web UI 容器作为 deployment

部署到pod默认的命名空间中。如果你想要将 YuniKorn 部署到另外一个命名空间，你可以进行如下操作：

xxxxxxxxxx
kubectl create namespace yunikorn-ns
helm install ./yunikorn --namespace yunikorn-ns

如果你不想使用 helm charts，你可以按照这个教程（点击我）一步一步的部署YuniKorn。

使用YuniKorn Scheduler运行workloads

和默认的 K8s Scheduler不同，为了更好的支持批处理YuniKorn 有 application的概念 。下面有几种方式使用YuniKorn scheduler运行批处理:

  • 在pod的规范中添加标签applicationId 和 queue。
  • 具有相同applicationId的pod将被视为来自1个应用程序的任务。

添加条目示例如下：

xxxxxxxxxx
  labels:
    applicationId: "MyOwnApplicationId"
    queue: "root.sandbox"

下一节中提供的所有示例都已设置了标签。 queue名称必须是配置中的已知队列名称，否则 YuniKorn scheduler 将会拒绝未知队列名称的pod。

运行一个简单的示例应用

所有的示例部署都可以在examples目录下找到。在 README文件中列出了所有的示例，这里并没有展示所有的示例，更多详细信息的可以在README中找到。

一个基于标准nignx镜像的单节点pod：

xxxxxxxxxx
kubectl create -f examples/nginx/nginx.yaml

一个简单的睡眠任务示例：

xxxxxxxxxx
kubectl create -f examples/sleep/sleeppods.xml

这些示例的文件可以在 README nignx 和 README sleep章节找到。

运行一个Spark应用

K8s对 Apache Spark 的支持不是所有版本的一部分。你必须使用内置 K8s 支持的最新版本的 Apache Spark。examples/spark 目录中包含了 Apache Spark driver 和 executor的pod模板文件，如果你想要使用YuniKorn scheduler去运行Spark on K8s任务，你可以使用它们。

  • 从spark官网获取最新的spark-3.0.0-preview (只有最新版本支持特定的 pod 模板)。
xxxxxxxxxx
$ wget http://mirrors.tuna.tsinghua.edu.cn/apache/spark/spark-3.0.0-preview/spark-3.0.0-preview-bin-hadoop2.7.tgz

或者获取spark-3.0.0版本以上的源码，自己进行编译

xxxxxxxxxx
$ wget hhttp://mirrors.tuna.tsinghua.edu.cn/apache/spark/spark-3.0.0-preview/spark-3.0.0-preview.tgz
# 解压后编译
$ mvn clean package -Pyarn -Phadoop-2.7 -Dhadoop.version=2.7.0 -Phive -Pkubernetes -Phive-thriftserver -DskipTests
  • 制作spark镜像
xxxxxxxxxx
$ ./bin/docker-image-tool.sh -r yunikorn build
  • 运行 spark submit
xxxxxxxxxx
/opt/spark/bin/spark-submit --master k8s://http://localhost:8001 \
  --deploy-mode cluster --name spark-pi \
  --class org.apache.spark.examples.SparkPi \
  --conf spark.executor.instances=1 \
  --conf spark.kubernetes.container.image=yunikorn/spark:latest \
  --conf spark.kubernetes.driver.podTemplateFile=examples/spark/driver.yaml \
  --conf spark.kubernetes.executor.podTemplateFile=examples/spark/executor.yaml \
  local:///opt/spark/examples/jars/spark-examples_2.12-3.0.0-preview.jar

Spark使用自己的版本的应用程序ID标记被称作spark-app-id。必须使用此标签才能将pod识别为特定的spark pod。

  • examples/spark/driver.yaml
  • examples/spark/executor.yaml

当你在K8s上使用pod模板运行spark，spark-app-id 被视为applicationId。在 README spark 章节中提供了运行spark程序的脚本和yaml文件。

亲和性调度

yunikorn调度器在k8s上使用predicates（谓词）支持亲和性与反亲和性调度：

xxxxxxxxxx
kubectl create -f examples/predicates/pod-anti-affinity-example.yaml

deployment（部署）保证2个pod不会在同一个节点上。如果这个yaml部署在一个单节点集群中，只会有一个启动，而其它一个会处于pending状态。更多关于在predicates的亲和性和反亲和性调度请查看README predicates章节。

Volume示例

这里有三个有效的示例。NFS示例在 docker desktop 中不起作用，需要 minikube。EBS需要 K8s 集群运行在AWS(EKS)上。关于volume示例更多详细的介绍在 README Volumes章节。

注意：所有示例都将在挂载volume上一个名为“dates.txt”的文件中生成一个无止境的数据流。这可能会造成磁盘溢出，所以执行时间应该被限制。

本地Volume

  • 创建本地volume 和 volume 声明
xxxxxxxxxx
kubectl create -f examples/volume/local-pv.yaml
  • 使用本地volume创建pod
xxxxxxxxxx
kubectl create -f examples/volume/pod-local.yaml

NFS volume

  • 创建 NFS 服务
xxxxxxxxxx
kubectl create -f nfs-server.yaml
  • 更新pod的YAML文件，用获取NFS服务的IP地址替换示例IP：
xxxxxxxxxx
kubectl get services | grep nfs-server | awk '{print $3}'
  • 使用NFS volume创建pod
xxxxxxxxxx
kubectl create -f pod-nfs.yaml
